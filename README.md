# triagefixgraph-ai-demo — Agent Memory Context Graph

[![Neo4j Labs](https://img.shields.io/badge/Neo4j_Labs-blue?logo=neo4j)](https://neo4j.com/labs/)

AI agent conversation tracking, entity extraction, memory management, and tool usage analysis

> AI-powered Agent Intelligence

Built with [create-context-graph](https://github.com/neo4j-labs/create-context-graph) · Agent Framework: **OpenAI Agents SDK**

> **Neo4j Labs Project** — Generated by create-context-graph, a [Neo4j Labs](https://neo4j.com/labs/) project. Maintained by Neo4j staff and the community, but not officially supported.

## Quick Start

```bash
# Install dependencies
make install

# Start Neo4j (Docker)
make docker-up

# Seed sample data
make seed

# Start the application
make start
```

- **Backend:** http://localhost:8000
- **Frontend:** http://localhost:3000
- **Neo4j Browser:** http://localhost:7474

## Domain: Agent Memory

### Base POLE+O Entities

These entity types are shared across all domains using the Person-Organization-Location-Event-Object model:

- **Person** — PERSON
- **Organization** — ORGANIZATION
- **Location** — LOCATION
- **Event** — EVENT
- **Object** — OBJECT

### Domain-Specific Entities
- **Agent** — OBJECT
- **Conversation** — EVENT
- **Entity** — OBJECT
- **Memory** — OBJECT
- **ToolCall** — EVENT
- **Session** — EVENT

### Demo Scenarios

#### Agent Introspection
- "What does agent Alpha remember about the user's project preferences?"
- "Show me all entities that agent Alpha has extracted this week"
- "Which tools does agent Alpha use most frequently?"

#### Conversation Analysis
- "Summarize the key topics discussed across all conversations today"
- "Which conversations were escalated and why?"

#### Memory Management
- "What are the most important memories stored by agent Alpha?"
- "Show me the entity graph extracted from recent conversations"
- "Which memories haven't been accessed in over 30 days?"

## Agent Tools

The AI agent has access to the following domain-specific tools:

- **search_agent** — Search for agents by name, model, or status
- **conversation_history** — Get conversation history for a specific agent
- **memory_recall** — Search agent memories by content or referenced entities
- **tool_usage_stats** — Analyze tool usage patterns for an agent
- **entity_graph** — Explore the entity relationship graph extracted from conversations
- **list_agents** — List agent records with optional limit
- **get_agent_by_id** — Get a specific agent by ID with all connections
- **execute_cypher** — Run ad-hoc Cypher queries
- **get_schema** — View the graph schema

```bash

```

| Variable | Description |
|----------|-------------|
| `NEO4J_URI` | Neo4j connection URI |
| `NEO4J_USERNAME` | Neo4j username |
| `NEO4J_PASSWORD` | Neo4j password |
| `ANTHROPIC_API_KEY` | Anthropic API key for the AI agent |
| `OPENAI_API_KEY` | OpenAI API key (for embeddings) |
| `SESSION_STRATEGY` | Memory session strategy (per_conversation, per_day, persistent) |
| `BACKEND_PORT` | Backend port (default: 8000) |
| `FRONTEND_PORT` | Frontend port (default: 3000) |

## Troubleshooting

### Neo4j Connection Errors

**"Neo4j is unavailable" on all endpoints**
- Verify your `.env` file has correct `NEO4J_URI`, `NEO4J_USERNAME`, and `NEO4J_PASSWORD`
- Run `make test-connection` to validate credentials
- Run `make docker-up` first, then wait a few seconds for Neo4j to start

**Extra variables in `.env`**
- The backend accepts extra environment variables, so standard Neo4j Aura `.env` exports (with `NEO4J_DATABASE`, `AURA_INSTANCEID`, etc.) work without editing

### Backend Issues

**Port already in use**
- Change `BACKEND_PORT` in `.env` (default: 8000)
- Kill existing process: `lsof -ti:8000 | xargs kill`

**API key not set**
- Set `ANTHROPIC_API_KEY` or `OPENAI_API_KEY` in `.env` depending on your agent framework

### Frontend Issues

**Build errors after `npm install`**
- Delete `node_modules` and `package-lock.json`, then run `npm install` again
- Ensure Node.js 18+ is installed
